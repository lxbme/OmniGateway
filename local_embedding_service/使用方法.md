# 本地编译

参考 [此处](USE_GUIDE.md)配置较为复杂, 推荐直接使用 docker 镜像，或者按照以下步骤本地编译：
本地编译需要安装 cmake 和 protobuf 编译器，且 protobuf 版本需要与项目中使用的版本一致，否则可能会遇到编译错误。

## 1. 清理历史构建
```bash
cd /your_path/go_projects/OmniGateway/local_embedding_service
rm -rf build
```

## 2. 重新构建
```bash
cd /your_path/go_projects/OmniGateway/local_embedding_service
mkdir -p build && cd build
cmake -DUSE_PREGENERATED_PROTO=OFF -DCMAKE_BUILD_TYPE=Release ..
cmake --build . -j
```
- 推荐 `USE_PREGENERATED_PROTO=OFF`：避免本地系统 protobuf 版本不一致。
- 如果你确实要用 pre-generated code：
  ```bash
  cmake -DUSE_PREGENERATED_PROTO=ON -DCMAKE_BUILD_TYPE=Release ..
  cmake --build . -j
  ```

## 3. 启动服务

### 3.1 Mock 模式（默认，推荐开发测试使用）
```bash
cd /your_path/go_projects/OmniGateway/local_embedding_service/build
./embedding_server
```

## 4.测试服务
新开一个终端，执行以下命令测试服务是否正常启动：
```bash
./test_all_features.sh
./check_embedding_service.sh
./test_onnx_fallback.sh
```

说明：
- `test_onnx_fallback.sh` 用于验证 Issue #16 的 fallback 场景：当 ONNX 初始化失败时，服务自动回退到 Mock 后端并保持可用。
- `test_all_features.sh` 已包含 Issue #17 的 Rerank 接口验证。

## 5. 手动验证 Rerank 接口
```bash
grpcurl -plaintext -import-path proto -proto proto/embedding.proto \
  -d '{"queries":[{"query":"hello world","documents":["hello world doc","irrelevant text","world hello again"],"top_k":2}]}' \
  localhost:50051 embedding.EmbeddingService/Rerank
```

# 使用docker

如果不想本地编译，可以直接使用 docker 镜像，步骤如下：
使用前请确保你已经安装了 Docker，且 Docker 服务正在运行。

```bash
cd .. # 进入 local_embedding_service 上级目录，确保 docker-compose.test.yml 在当前目录下
docker-compose up -d embedding-service
```
- 这会拉取预编译的 embedding-service 镜像并启动服务。
- 启动后同样可以使用 `./test_all_features.sh` 和 `./check_embedding_service.sh` 来验证服务是否正常运行。
